Yes, it is true that the template's HTML preview does not set a relevant response header (the plain preview does). This is because you normally define the character set in HTML using a meta tag (<meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />) which should be enough. If you omit the meta tag, the preview relies on either the server setting a Content-Type header (which might include the wrong chracter set) or on the client's default character set.
